# RPC - DISTRIBUTED MOVIE MANAGEMENT SYSTEM (GRPC + MONGODB ATLAS)

## PROJECT DESCRIPTION

This project is a high-level distributed application composed of a TypeScript daemon server and an interactive Python client. The system allows complete management (CRUD) of a movie catalog using the "sample_mflix" dataset hosted on MongoDB Atlas. 

The architecture utilizes the gRPC (Google Remote Procedure Call) pattern operating over HTTP/2. Delegating network complexity, binary serialization (Protocol Buffers), and routing to the gRPC engine allows the system to focus entirely on business rules. The project was also containerized via Docker to guarantee immutability and isolated execution on any machine without the need to manage local dependencies, such as node modules required to run the TypeScript server.

---

### TYPESCRIPT SERVER

The server acts as the processing, telemetry, and persistence core of the system. It awaits remote procedure calls (RPC), executes fail-fast semantic validations, processes mutations on MongoDB Atlas, and returns the data to the client. It also features an interceptor for traffic observability and latency calculation.

Features:
- Implementation of the gRPC service (MovieService).
- Protection against memory leaks and strict typing (with ObjectId.isValid).
- Mutation using the `optional` fields of Proto3.
- Interceptor for network tracing.

Libraries Used:
- `@grpc/grpc-js` and `@grpc/proto-loader`: HTTP/2 routing engines and .proto contract readers.
- `mongodb`: Official driver for cloud data manipulation.
- `dotenv`: Secure environment variable injection.
- `tsx`: Direct execution TypeScript interpreter.

### How to Run:

#### Option A: Using Docker (Recommended)

Prerequisites
* Docker and Docker Compose installed on your machine.
* The `.env` file properly configured inside the `serverTS` directory (with your `MONGO_URI`).


At the root of the project (where docker-compose.yml is located), run the command to build and orchestrate the infrastructure in the background:
```bash
docker compose up --build -d
```

To monitor server telemetry in real-time, open a terminal tab and run:
```bash
docker logs -f rpc_server
```

#### Option B: Locally
Ensure you have node and npm libraries installed.
```bash
node -v
npm -v
```
If the commands are not available, you will need to properly install the base libraries.

After the base libraries installation procedure, install the project dependencies in the TypeScript server directory (serverTS) with:
```bash
npm install
```
Rename the env.example file to .env and insert your MONGO_URI with the uri provided by mongoDB Atlas.

Start the server in development mode with the command:
```bash
npm run dev
```

### PYTHON CLIENT

The client is an interactive Command Line Interface (CLI). It manages Stubs generated by the gRPC compiler, invoking methods on the server as if they were local functions. It features a syntactic analyzer to avoid malformed commands and provides ergonomics to the operator (e.g.: displaying previous state before an UPDATE).

Available commands for the client:
| Command | Description |
|---------|-----------|
|`LSACT` | Searches and lists (maximum batch of 50) movies where the specified actor participated in the cast. |
|`LSCATG` | Filters and lists movies by exact category (e.g.: action, drama). Features real-time validation. |
|`READ` | Extracts and displays the full record of a movie through its physical ID. |
|`CREATE` | Initiates the insertion flow. The system guides the operator by displaying valid categories. |
|`UPDATE` | Executes an invisible parallel call to fetch the current state of the movie, displays it, and performs the mutation of the chosen field. |
|`DELETE` | Permanently removes a document from the database via ID validation. |
|`EXIT` | Disconnects from the HTTP/2 channel and closes the terminal. |


Libraries Used:
- `grpcio`: Main RPC communication engine over HTTP/2.
- `grpcio-tools`: Native compiler to generate Python structures from the .proto contract.

### How to Run:

#### Option A: Using Docker (Recommended)
Ensuring the infrastructure has already been raised with docker compose up, attach your physical terminal to the client interface by running:
```bash
docker attach rpc_client
```
Upon executing the command above, the client will already be working. You will need to press the `Enter` key again for the shell indicator `rpc-cli>` to appear.

`Note`: Using SIGINT (Ctrl+C) to close the client will shut down the docker container responsible for it. To disconnect from the client without killing the container, use the internal EXIT command or the sequence Ctrl+P, Ctrl+Q.

If you closed the container responsible for the client with Ctrl+C, it is possible to execute the following command to reactivate it:

```bash
docker start rpc_client
```

#### Option B: Locally
Access the clientPY directory and create the virtual environment: 
```bash
python3 -m venv venv
```
Then: 
```bash
source venv/bin/activate
```
Install protocol dependencies: 
```bash
pip install grpcio grpcio-tools
```
If the contract changes, recompile the stubs: 
```bash
python3 -m grpc_tools.protoc -I. --python_out=. --grpc_python_out=. proto/movies.proto
```
At the end of the main procedures, execute the interactive panel: 
```bash
python3 main.py
```
